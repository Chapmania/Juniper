74  /*
75  In Bluetooth GATT, Characteristics are key-value pairs, collected into "Services",
76  that represent the raw functionality of your device. Characteristics support
77  both reading from and writing to, as well as notification events when values
78  change.
79  The following lookup table provides all of the standard, registered Characteristics
80  as of September 1st, 2017.
81  The supported data types are very primitive:
82  - Strings of up to 20 bytes in length, and strictly in ASCII encoding
83  - Integers in 1, 2, or 4 byte lengths
84  - Floats in 4 or 8 byte lengths
85  - "Auto", which is basically just a String
86  Characteristics can also have Descriptors, which provide more metadata about
87  the value, e.g. friendly, readable descriptions when more than one "ANALOG"
88  type characteristic is included in a Service.
89  Theoretically, Services have a required set of Characteristics, though I've
90  never seen this enforced. Characteristics are supposed to also have a proscribed
91  data type and length, but this is not enforced by the Bluetooth stack, either,
92  and is really only an aid to consuming application. A lot of apps assume any
93  particular Service will only ever contain one of any type of Characteristic,
94  but this is not expressed in the spec and only comes about from folks who don't
95  understand how Descriptors work to differentiate between multiple Characteristics
96  of the same type. Characteristics in practice are nothing but an arbitrary ID
97  coupled with an arbitrary set of metadata and an arbitrary value. This makes it
98  possible for vendors to make up their own Characteristics whenever they want.
99  Just be sure to stick to the higher value ranges so you don't step on anyone
